## 1. ゲームコンセプト

**LiDARで自分の足を追跡し、M5Stickを付けた「鎌」を実際に振り回して戦う、全身で遊ぶアクションゲーム。**

プレイヤーは床に映る戦場の中を歩き回り、鎌を振って敵を斬り、立ち止まって弾を跳ね返す。コントローラのボタンではなく「歩く・振る・止まる」という体の動きそのものが操作になる。

体験の核は **「動き回る爽快感」**。だから敵も攻撃も全部「足元の床」に映る。

## 2. 物理セットアップ（プレイ環境）

```
            ┌─────────────────┐
            │   壁面プロジェクタ   │ … スコア・HP・時間（情報表示）
            └─────────────────┘
                     ▲
                     │ 顔を上げて確認
      🧍 プレイヤー（鎌を持つ）
       │  足の位置をLiDARが検出
       ▼
   ┌───────────────────────┐
   │   床面プロジェクタ（メイン）   │ … 敵・弾・攻撃・自分の位置（ゲーム本体）
   └───────────────────────┘

   手に持つ鎌 → M5StickC → USB/シリアル → PC
   足の位置  → LiDAR → footpoint.txt → PC
```

- **床（メイン）**: 真上または斜めから床に投影。ここで戦う。
- **壁（サブ）**: 正面の壁に投影。スコアなどの情報だけ。
- **鎌**: M5StickC（IMUセンサー内蔵の小型マイコン）を取り付け、振りの動きを検出。
- **LiDAR**: 光で距離を測り、プレイヤーの足の位置を割り出す。

## 3. システム構成（データの流れ）

```
[M5Stick(鎌)]──シリアル115200──┐
  加速度3軸+ジャイロ3軸          ├──→ [ゲーム(main.cpp)]──→ [床ウィンドウ] 戦場
                                │         ↑ 50fpsで更新      └→ [壁ウィンドウ] UI
[LiDAR]──footpoint.txt──────────┘
  足の座標(x,y)
```

| 入力源 | 渡すもの | 渡し方 | ゲーム内の変数 |
|--------|---------|--------|---------------|
| M5Stick | 加速度・ジャイロ6値 | シリアル通信(USB) | `val[0]`〜`val[5]` |
| LiDAR | 足の座標（点群） | `footpoint.txt`を毎フレーム読む | `footPos[]`, `footNum` |

> ⚠️ 2つの入力は**別系統**。M5Stickはシリアル直結、LiDARはファイル経由。  
> このため起動順序（LiDARを先に）が重要。

## 5. ゲームの状態遷移

```
 STATE_START ──鎌を振る──→ STATE_RULE ──鎌を振る──→ STATE_PLAY
  (タイトル)                (ルール説明)              (ゲーム本編)
                                                          │
                              ┌───────────────────────────┤
                          HPが0                        ボス撃破
                              ▼                            ▼
                      STATE_RESULT(GAME OVER)      STATE_RESULT(CLEAR)
                              └──────鎌を振る──────→ STATE_START へ戻る
```

メニューの送りも「鎌を振る」動作（`val[3]` か `val[4]` が大きく動く）で行う。連打防止に `sceneTransitionTimer` を使う。

## 6. 操作とアクション

| 操作 | 検出方法 | 効果 |
|------|---------|------|
| **歩く** | LiDARの足座標 `footPos[0]` の移動 | プレイヤーが移動。移動方向に体が向く |
| **縦振り** | ジャイロ `val[3]`（Pitch）が閾値超え | 前方への強攻撃。射程長・ダメージ80 |
| **横振り** | ジャイロ `val[4]`（または横持ち時の`val[3]`）が閾値超え | 周囲への薙ぎ。半径150・ダメージ20 |
| **防御（構え）** | その場で静止＋鎌を止める | シールド展開。前方45度の弾を反射 |
| **オートエイム** | 攻撃した瞬間に自動発動 | 200以内に敵がいればその方向を自動で向く |

### 攻撃判定の工夫（最新版）
- **振り抜き・戻り判定**: 一度振ったら、`val[3]`と`val[4]`が小さくなるまで次の攻撃を受け付けない（`canAttack`）。1回の振りが何度も誤判定されるのを防ぐ。
- **優位軸判定**: 縦と横のどちらが「明らかに大きいか」（`dominance_ratio`）で振りの種類を決める。斜め入力の誤爆を防ぐ。
- **横持ち対応（`isHeldSideways`）**: 加速度`val[0]`で鎌が寝ているか検知し、寝ているときの縦回転は「横振り」と解釈する。

### 防御・反射のしくみ
```
立ち止まる → isDefending = true → シールド表示
   ↓
敵弾が前方45度以内＆距離85以内に来る
   ↓
反射成功！ → 弾が「味方の弾」に変化、速度1.5倍で跳ね返る、発光演出
   ↓
跳ね返した弾が敵に当たる → 敵に50ダメージ
```

## 7. ゲーム進行（WAVE構成）

| WAVE | クリア条件 | 敵の攻撃 | 設計意図 |
|------|-----------|---------|---------|
| **WAVE 1** | 30秒間 生き残る（その間に倒しまくる） | **なし** | チュートリアル。鎌の振り方を練習。スコア稼ぎ |
| **WAVE 2** | 雑魚を4体倒す | **あり** | 弾を反射しながら戦う練習 |
| **WAVE 3** | ボスを倒す | **あり（3パターン）** | 全アクションを総動員 |

各WAVEクリア時、**紙吹雪が5秒間舞う演出**（`waveClearTimer = 250`）が入ってから次へ進む。

- WAVE1: 常に敵4体を維持（倒すと補充）。時間切れでクリア。
- WAVE2: 常に敵2体を維持。累計4体撃破＋画面上が空になったらクリア。
- WAVE3: ボス1体。撃破でゲームクリア。

## 8. 敵とボス

### 雑魚敵
- HP: WAVE1=30 / WAVE2=80
- 8か所の出現位置からランダムに登場（位置が被らないよう調整）
- WAVE2以降、距離800以内のプレイヤーに通常弾を撃つ（約4.8秒間隔）

### ボス（WAVE3）
- HP: **2000**
- 画面内をランダムな目標へ向かって移動
- ランダムで防御状態（弾が効かない）
- **3つの攻撃を順番に繰り返す**：

```
パターン0: 3WAY通常弾   … 扇状に3発（やや速い）
   ↓
パターン1: ビーム       … 100フレーム溜め → 高速の単発ビーム
   ↓
パターン2: 20WAY爆発弾   … 100フレーム溜め → 全方向に20発ばらまき
   ↓ (0に戻る)
```

溜め中は本体が点滅し、専用画像（溜めモーション）に切り替わる。

## 9. 弾・アイテム・相互作用

### 弾の種類とダメージ（プレイヤーが受ける）
| 種類 | ダメージ | 見た目 |
|------|---------|--------|
| 通常弾（雑魚・ボス） | 10 | 小さい弾 |
| ビーム（ボス） | 30 | 大きい横長 |
| 投石/爆発弾（ボス） | 20 | 岩（3種ランダム） |

### アイテム
- 出現: 約20秒ごと＋敵撃破時10%でドロップ
- **回復**: HP+50（上限100）／緑フラッシュ演出
- **攻撃バフ**: 10秒間ダメージ1.5倍／オレンジフィルター（残り3秒で点滅）

### プレイヤー
- HP: 100（0でゲームオーバー）
- 被弾時: 画面が揺れる＋赤フラッシュ（`playerDamageTimer`）

## 10. スコアとランク

```
撃破スコア:  雑魚 = +100 / ボス = +1000

クリア時のランク計算:
  ランクスコア = 獲得スコア × (残りHP ÷ 100)

  S : 1500以上   ← 高スコア＆高HP維持
  A : 1000以上
  B :  500以上
  C : それ未満
  D : ゲームオーバー（HP0）
```

「スコアを稼ぐ」だけでなく「HPを残す（＝うまく避ける・防ぐ）」ほど高ランク。**攻めと守りの両立**を促す設計。

## 14. 今後の発展（提案）

「動き回る爽快感」というこのゲームの強みを伸ばす方向で考える。短期＝すぐ試せる、中期＝作り込み、研究的＝研究室（画像処理・CG）で深められるテーマ。

### 短期（すぐ試せる・体験が良くなる）
- **振りの速さでダメージ可変**: 角速度（`val[3]`/`val[4]`の大きさ）が大きいほど高ダメージ。鎌を「強く振る」ことに意味が出て爽快感が増す。
- **コンボ表示**: 連続ヒットで倍率上昇＋画面に「COMBO x3」。手を止めない動機になる。
- **溜め斬り**: 鎌を一定時間構えてから大振りすると強攻撃。攻めと守りの中間アクション。

### 中期（ゲームとしての深み）
- **壁面の敵を反射で倒す**（あなたのアイデア）: 空中の敵は鎌が届かず、床の弾を跳ね返して倒す。「防御＝攻撃手段」になり、2画面を使う意義が明確になる。
- **足の軌跡をギミック化**: 通った床に効果（炎の道・回復ゾーン）。LiDARで軌跡を取っている強みを活かす。
- **動的難易度調整**: プレイヤーのHPや撃破ペースを見て敵の数・速度を自動調整。誰でも楽しめる。
- **複数人プレイ**: LiDARは複数の足を検出できるので、協力プレイや対戦に拡張できる。

### 研究的（研究室で深められる・卒研ネタにも）
- **センサーフュージョン**: LiDAR（位置）とIMU（姿勢）を統合し、より正確に全身の動きを推定する。「2つのセンサーの良いとこ取り」は立派な研究テーマ。
- **LiDAR点群からの姿勢推定**: 足だけでなく、点群の形から体の向き・しゃがみ・ジャンプを推定（画像処理・点群処理の応用）。今の「移動方向から向きを推測」を根本的に改善できる。
- **プロジェクションマッピングの精度向上**: 床への投影を正確にキャリブレーション。実空間とゲーム座標を高精度に対応づける（CG・幾何の研究）。
- **CG表現の強化**: シェーダーやポストエフェクトで、斬撃・爆発・反射をより派手に。OpenGLの発展学習にもなる。

> これらは全部一度にやらない。まず**土台（M5Stickが正しく動く・向きが歩行に追従する・コードが整理されている）**を完成させてから、面白さを足していく。
